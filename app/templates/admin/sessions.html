Configure sources

A provider is one railway / transit operator that contributes data to this session. Each provider can publish a timetable (GTFS or NeTEx) plus optional GTFS-RT live updates, minimum-connection-time table, and stations CSV. Add as many as you need (SNCF + IDFM + Trenitalia + …) — they're combined into one OTP graph at build time, with cross-operator transfers auto-generated between stops within walking distance.

Providers

Must cover every region your providers reach. France-wide PBF + IDFM Paris GTFS is fine; IDF-only PBF + nationwide SNCF GTFS will fail to snap lat/lon coordinates outside Île-de-France. (Coordinate-search fallback to station names is in the journey UI — see v0.1.7.)

Filter runs in the otp-build container via osmium-tool, before OTP parses the PBF. It cuts the data OTP has to hold in heap, not the routing accuracy — transit-focused keeps all roads + railways + public-transport amenities. Default for new sessions: transit-focused.

Upload a file

Manual alternative to Refresh sources. Files are dispatched into the right inbox subfolder.

Build & Promote

Rebuild queues an OTP graph build. The worker picks up pending jobs every 15 s after the debounce window. Build duration depends on the bundle: ~10 min for IDF, ~30–60 min for France-wide. On success the session advances to graph_built.
Promote moves graph_built → serving and asks the worker to docker compose up + nginx -s reload so the new otp-{{ s.id }} container becomes routable at /otp/{{ s.id }}/.
