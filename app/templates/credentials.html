My API credentials

Save the API keys / bearer tokens / basic-auth pairs you use to access authenticated provider feeds (SNCF, SBB, Trenitalia, etc.). Once saved, pick from the dropdown next to any provider URL on a session you manage. Secrets are encrypted at rest with AES-256-GCM, derived from the server's JWT_SECRET. They're never shown back to anyone, including you — if you forget a value, rotate (re-enter) it.

Where do my saved credentials get used?
A credential here is the auth value (token / user:pass). To actually use it, the system needs to know which URL to send it with. That happens in one of two places:
  • NAP catalogues at /admin/nap-catalogues — pair your credential with a NAP API endpoint URL. The Import-from-NAP modal then applies it server-side. Platform-admin only.
  • Provider URLs in a session config — each provider's timetable, gtfs_rt, mct, and stations_csv URL field has an optional credential dropdown next to it.
One credential can be reused in multiple places — that's why URL and credential are kept separate.
